1. Introduction¶

The purpose of this note is to show how one can reproduce some of the statistical analysis of Lee et al., using data made accessible by the authors of the paper on Gene Expression Ombibus (GEO), accession code GSE34721.

The reason we decided to replicate the analysis presented in the paper was two-fold:

  1. We wanted to convince ourselves that the analysis is free of substantial errors.
  2. We would like to apply their approach to a similar, previously unpublished dataset.

The added benefit of this replication is that we make our pipeline publically accessible, and freely usable to other researchers in the field.

Among other claims, Lee et al. propose the following:

  1. The length of the trinucleotide repeat specific to Huntington Disease (HD) leaves a discernible pattern in the mRNA profiles obtained from diseased and healthy patients, via lymphoblastoid cell lines, and measured using microarray technology.
  2. The strength of this pattern is positively correlated with the length of the repeat.
  3. The pattern can be used to construct a simple statistical model, which can predict the length of the repeat from the mRNA profile significantly better than a random predictor.
  4. The quality of prediction is modest, with an unoptimised model able to explain only about a fifth of repeat length variance.

For details specific to each of this claims, in particular the exact type of cell line and microarray chip, please see the original study.

We believe that all of these claims are well supported by the mRNA expression and repeat length retrieved from one of the data files we accessed on GEO.

In this note we do not attempt to carefully verify other aspects of the study, among others:

  1. How the patients were sampled & assigned to various batches.
  2. How the raw data were normalised & the Series Matrix file constructed.

3.3 Data pre-processing¶

The Matrix Series file uses the convention of representing patients as columns and features as rows in a tab-separated file format.

Lines beginning with the exclamation mark are meta-data. All other lines are RMA-normalised mRNA profiles.

The meta-data include, among others:

  1. sample status ("discovery set" or "replication set")
  2. sample's GEO accession (e.g. GSM853700, not the same as the accession number of the whole dataset)
  3. patient's sex (M or F)
  4. batch number (A-E, or R1-R6)
  5. repeat length (an integer value)

For the purpose of this replication study, we are only interested in the samples from the discovery set. Technical replicates, R1-R6 are used in the original paper to propose claims not tested in this replication study.

import random
def choose_training_indicies(data, no_training_samples):
    indices = list(range(data.shape[1]))
    random.shuffle(indices)
    training_indices = indices[:no_training_samples]
    test_indicies = indices[no_training_samples:]
    return training_indices, test_indicies

def order_indicies(allele, data):
    r2s = []
    for i in range(data.shape[0]):
        gene_r, _ = pearsonr(allele, data[i])
        # Note: we have to store both the gene and the gene index, to recover the indices after sorting.
        r2s.append((gene_r ** 2, i))
    r2s.sort()
    # Note: this reverses the list, assuring decreasing order.
    r2s = r2s[::-1]
    return r2s

8. Combining defined utility functions into a single simulation_step function¶

def simulation_step(training_size=97, no_probes=500, n_components=10):
    training_i, testing_i = choose_training_indicies(probeset_data, training_size)
    training_fold, testing_fold = split_data(probeset_data, training_i, testing_i)
    modal_allele_training = [modal_allele[i] for i in training_i]
    modal_allele_testing = [modal_allele[i] for i in testing_i]
    
    top_500 = order_indicies(modal_allele_training, training_fold)[:no_probes]
    training_set = choose_top_genes(top_500, training_fold)
    testing_set = choose_top_genes(top_500, testing_fold)

    plsr = PLSRegression(n_components=n_components)
    
    plsr.fit(training_set.transpose(), cv(modal_allele_training))
    
    results = plsr.predict(testing_set.transpose())
    results = [i[0] for i in results]

    return modal_allele_testing, results
